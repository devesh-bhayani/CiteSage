# Subword Tokenization: BPE, WordPiece, and SentencePiece

Tokenization is the process of converting raw text into a sequence of discrete units — tokens — that a language model's vocabulary and embedding layer can consume. Modern NLP systems almost universally use **subword tokenization**, which sits between word-level and character-level tokenization.

## Why Not Word-Level or Character-Level?

Word-level tokenization (splitting text on whitespace/punctuation into whole words) requires a vocabulary large enough to cover essentially every word form the model might encounter, yet still fails on any word not seen during vocabulary construction — producing an out-of-vocabulary (OOV) token that discards all information about the word. Character-level tokenization eliminates the OOV problem entirely (any text can be built from a small fixed alphabet) but produces very long token sequences for a given piece of text, which is computationally expensive for self-attention (whose cost grows with the square of sequence length) and makes it harder for the model to learn word-level and higher-level semantic patterns directly. Subword tokenization is a middle ground: common words remain single tokens, while rare or unseen words are decomposed into a small number of frequent subword pieces, giving both a bounded vocabulary and full coverage of arbitrary text with no true OOV tokens.

## Byte-Pair Encoding (BPE)

Byte-Pair Encoding builds its subword vocabulary via a simple, data-driven merge procedure. It starts with a base vocabulary of individual characters (or bytes), and represents the training corpus as sequences of these base units. It then repeatedly finds the most frequent adjacent pair of symbols in the corpus and merges that pair into a new single symbol, adding it to the vocabulary. This merge-and-repeat process continues for a fixed number of merges (or until a target vocabulary size is reached), with each merge recorded in an ordered merge list. At inference/tokenization time, the same ordered list of merges is applied greedily to new text, splitting it first into base characters and then repeatedly applying the highest-priority applicable merge, until no more merges apply. Because merges are learned from frequency statistics, common words and common subword fragments end up as single tokens, while rare words are split into a handful of more frequent pieces. GPT-2 and many subsequent models use a byte-level variant of BPE, where the base vocabulary is the 256 possible byte values rather than Unicode characters, guaranteeing that literally any input text (any language, emoji, or malformed encoding) can be tokenized without ever needing an unknown-token fallback.

## WordPiece

WordPiece, used by BERT, follows a very similar merge-based construction to BPE, but with one key difference: instead of merging the single most *frequent* adjacent pair at each step, WordPiece merges the pair that maximizes the likelihood of the training data under a simple language model — roughly, the pair whose merge most increases how "surprising" it would be if the two symbols occurred separately versus as one unit. In practice this tends to produce similar-quality vocabularies to BPE, and the two are often treated as interchangeable in casual usage, though the underlying merge criterion differs. WordPiece also conventionally marks continuation subwords with a special prefix (BERT uses "##", so "playing" might become ["play", "##ing"]), which lets a detokenizer unambiguously reconstruct which subword pieces belonged to the same original word.

## SentencePiece

SentencePiece is not a distinct tokenization algorithm so much as a tokenizer *implementation* framework that can run either BPE or a probabilistic method called unigram language model tokenization, with one crucial design choice: it treats the input as a raw stream of Unicode characters, including whitespace, rather than pre-splitting on whitespace before applying subword merges. This makes SentencePiece language-agnostic — it works identically well for languages that don't use whitespace to separate words, such as Japanese or Chinese, where a pre-tokenization step based on whitespace would not make sense. Because whitespace is treated as an ordinary token-able character (SentencePiece conventionally replaces spaces with a special meta-symbol, commonly "▁"), tokenization and detokenization are fully reversible without needing separate rules for reinserting spaces between words.
